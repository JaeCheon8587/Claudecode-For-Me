# ATDD Design & Test — 설계 + 테스트 코드 작성 (Red 단계)

FRD를 입력받아 내부적으로 설계를 수행한 뒤, **바로 테스트 코드를 작성한다.** 중간 산출물 파일(설계서, 테스트 명세)을 생성하지 않는다 — 설계는 에이전트 내부에서만 수행하고, 최종 산출물은 테스트 코드뿐이다.

이 스킬의 산출물은 컴파일이 안 되는 것이 정상이다. 테스트에서 참조하는 DTO, Service, Controller 등이 아직 존재하지 않기 때문이다. 이것이 ATDD의 Red 단계다.

**프로덕션 코드는 절대 작성하지 않는다.**

---

## Phase 0: 입력 검증

1. `$ARGUMENTS`에서 FRD 파일 경로를 파싱한다.
2. FRD 파일을 읽고 필수 섹션이 있는지 확인한다: 기능명, API 스펙, 기능 시나리오.
   - 누락 시 에러 메시지를 출력하고 종료한다.
3. FRD 파일명에서 `{기능명}`을 추출한다 (예: `frd_로그인.md` → `로그인`).

---

## Phase 1: 코드 탐색 (제한적)

프로젝트의 기술 스택과 컨벤션을 파악한다. **최소한의 파일만 읽는다.**

읽을 수 있는 파일:
- 프로젝트 설정 파일 **1개** (`.csproj`, `package.json`, `build.gradle` 등) — 프레임워크, 의존성 파악
- 테스트 프로젝트 설정 파일 **1개** — 테스트 프레임워크 파악
- 기존 유사 기능 파일 **최대 2개** — 네이밍 컨벤션, 레이어 패턴 파악

**⛔ 금지**: 프로젝트 전체 탐색, 디렉토리 구조 재귀 탐색, 관련 없는 파일 읽기. 위 범위를 초과하는 탐색은 하지 않는다.

```
[Phase 1 완료] 코드 탐색 완료 — 설계를 시작합니다.
```

---

## Phase 2: 내부 설계 (파일 출력 없음)

FRD를 읽고 아래 항목을 **내부적으로만** 정리한다. 파일로 저장하지 않는다.

- **레이어 구조**: 프로젝트 컨벤션에 맞는 레이어명, 책임, 클래스명
- **DTO 정의**: FRD의 Request/Response 스펙에서 도출한 필드, 타입, 유효성 규칙
- **메서드 시그니처**: 각 레이어의 public 메서드 (구현 본문 없이)
- **의존성 맵**: 상위 → 하위 단방향 의존만 허용
- **테스트 케이스 목록**: FRD 시나리오를 Given/When/Then으로 변환
- **DEFERRED 항목**: FRD 미해결 항목 → skip 처리할 테스트 식별

### 설계 규칙

- FRD에 없는 기능을 추가하지 않는다.
- Phase 1에서 파악한 프로젝트 컨벤션(네이밍, 레이어 패턴, 디렉토리 구조)을 따른다.
- 미해결 항목은 임의로 결정하지 않고 `[DEFERRED]`로 표기한다.

```
[Phase 2 완료] 설계 완료 — 테스트 코드 작성을 시작합니다.
```

---

## Phase 3: 테스트 코드 작성

Phase 2에서 정리한 설계를 바탕으로 실행 가능한 테스트 코드를 작성한다.

### 작성 순서

1. **정상 흐름(Happy Path) 테스트부터 작성한다.**
2. 예외/엣지 케이스 테스트를 이어서 작성한다.
3. `[DEFERRED]` 테스트는 주석으로 남기고 skip 처리한다 — 삭제하지 않는다.

### 작성 규칙

- 테스트 프레임워크는 Phase 1에서 파악한 기술 스택에 맞게 선택한다 (xUnit for ASP.NET, Jest for NestJS, JUnit for Spring Boot 등).
- 테스트 파일은 **테스트 프로젝트에만** 생성한다. 프로덕션 프로젝트에 파일을 생성하지 않는다.
- Given/When/Then 구조를 코드 주석이나 describe/it 블록 이름으로 반영한다.
- 외부 의존성(DB, 외부 API)은 내부 설계의 인터페이스를 기반으로 모킹한다.
- FRD의 모든 시나리오 단계가 하나 이상의 테스트 케이스로 커버되어야 한다.

---

## Phase 4: 완료

테스트 파일 작성이 끝나면 아래를 출력한다:

```
[ATDD-DESIGN COMPLETE] 테스트 코드 작성 완료
- 생성된 파일: {테스트 파일 경로 목록}
- 테스트 케이스 수: {N}건 (DEFERRED 스킵: {N}건)
```

---

## Tone & Style

- 한국어를 기본 언어로 사용한다.
- 코드는 해당 기술 스택의 관례를 따른다.
- 이 스킬은 서브 에이전트에서 실행되므로 `AskUserQuestion`을 호출하지 않는다.
- 기존 테스트 코드가 있으면 그 스타일을 따른다.
